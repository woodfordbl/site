# Page commands

Page lifecycle commands are separate from canvas block commands (see [canvas-commands](./canvas-commands.md)). Block rendering on a page uses `BLOCK_SPECS` via `BlockTreeNode` / `BlockRenderer` (including checklist, divider, numbered list, and tabs blocks; see [block-types](../architecture/block-types.md)).

| Command | Meaning |
|---------|---------|
| `page.create` | New user page + `navigate` effect (`userPage: true` → `/p/$` splat for the allocated metadata slug); optional `parentId`, `insertAfterPageId` (sidebar order immediately after that sibling), `pageId`, `navigate: false`, `initialBlocks`, `icon`, `headerImage`, `databaseRowSource` (row-page materialization marker — the created page is hidden from the sidebar tree; [databases — Row pages](../architecture/databases.md#row-pages-virtual--copy-on-write)) |
| `page.update` | Update title/slug metadata via `page.persist` (no router navigate effect) |
| `page.delete` | Remove user pages (hard delete, also clears version-history snapshots) or hide shipped pages locally (`deletedAt` tombstone, snapshots kept); always keeps at least one page and never deletes home |
| `page.resetToRemote` | Full reset of one shipped page overlay — deletes local metadata + blocks (and version-history snapshots), restores bundled JSON on next read ([`resetPageToRemote`](../../src/lib/pages/reset-page-to-remote.ts)). Driven by site settings **Development** → **Reset page** and the sidebar row menu |
| `page.resetAllToRemote` | Clears all local pages, block shards, snapshots, and hint cookies; navigates home ([`resetAllToRemote`](../../src/lib/pages/reset-all-to-remote.ts)). Driven by site settings **Development** → **Reset all** |

Site settings **Refresh site content** is not a `page.*` command — it calls [`refreshSiteContent`](../../src/lib/pages/refresh-site-content.ts) directly, looping `resetPageToRemote` over only the content-stale overridden pages from [`useSiteContentUpdates`](../../src/hooks/use-site-content-updates.ts). See [author-dev-mode](../architecture/author-dev-mode.md) and [local-first-persistence](../architecture/local-first-persistence.md#hybrid-server--local-merge).
| `page.reposition` | Update `parentId`, `sidebarOrder`, and metadata `slug` (with descendant cascade); optional `appendPageLinkOnParent` appends a child `pageLink` on the parent canvas (nest drops only) |

Effects ([`PageEffect`](../../src/lib/canvas/effects.ts)): `page.persist` (insert/update metadata in `localPagesCollection`, blocks in `localBlocksCollection`; on `create: true`, [`purgeSlugTombstonesForUserPageCreate`](../../src/lib/pages/resolve-user-page-by-slug.ts) removes soft-deleted rows with the same slug + `parentId` before insert; slug renames cascade descendants — address-bar sync only when the caller passes `syncUrl: true` to [`persistPageMetadata`](../../src/lib/pages/persist-page-metadata.ts), e.g. title blur, with `{ userPage: true }` when merged `routeBy === "id"`), `page.reposition` ([`PageRepositionPlan`](../../src/lib/pages/reposition-page.ts) + optional `seed` / `parentSeed`, then [`persistPageReposition`](../../src/lib/pages/persist-page-reposition.ts) which calls `syncPageUrl` when the slug changes), `page.delete`, `navigate` (`slug` + optional `userPage` → [`pageNavTargetForUserPage`](../../src/lib/pages/slugify.ts) `/p/$` or [`pageNavTarget`](../../src/lib/pages/slugify.ts) with `replace: true` when `mode` omitted or `router`; `mode: "history"` → [`syncPageUrl`](../../src/lib/pages/sync-url.ts) with the same `userPage` flag). Inserts set `createdAt` and `updatedAt` once; updates bump `updatedAt` only. Metadata, settings, and reposition persists (and block commits) schedule a debounced page version-history snapshot ([local-first-persistence — Page snapshots](../architecture/local-first-persistence.md#page-snapshots-version-history)); the ⋯ menu's **Version history** action opens a read-only preview + restore overlay ([pages — Version history](../architecture/pages.md#version-history)), and its **Move to** picker ([`page-move-targets.ts`](../../src/lib/pages/page-move-targets.ts)) lists parent targets with each page's own icon. `/p/$` routes resolve the splat via [`resolveActiveUserPageBySlug`](../../src/lib/pages/resolve-user-page-by-slug.ts) ([`useResolvedUserPage`](../../src/hooks/use-resolved-page.ts)) — tombstones are not valid targets.

`page.create` accepts optional `parentId`, `insertAfterPageId`, `initialBlocks`, `icon`, `headerImage`, and `databaseRowSource` (threaded onto the `page.persist` effect and stored on the local page document; used by database row-page materialization — such pages never render in the sidebar tree). When `initialBlocks` is non-empty, those blocks are seeded instead of the default empty text row (used by sidebar **Duplicate page**). **Duplicate page** passes `insertAfterPageId` so the copy appears directly below the source row in the same parent scope, plus the source page's `icon` and `headerImage` (cover) — resolved from the local page document/shard when edited, else the shipped JSON via [`resolveSourceBlocksForPage`](../../src/lib/pages/resolve-source-page-blocks.ts) — so the copy keeps the same icon and cover. When set, the new page metadata slug is built from the parent path plus a slugified title segment. Duplicate segments are suffixed among **all siblings** via `allocateUserPageSlug` (`notes-2`, etc.). Creation is rejected when the parent is already at `MAX_PAGE_DEPTH`. Pass `navigate: false` to create a page without leaving the current canvas (used by slash **New Page**). Default title is `New Page`. Optional `pageId` lets callers pre-assign the id before inserting a matching `pageLink` block. Slash menu page rows are built by [`buildRootSlashMenuItems`](../../src/lib/canvas/slash-menu-list.ts) from the canvas editor's merged page list (filtered **New Page** + **Link To Page** trigger). **New Page** runs `page.create` with `navigate: false`, then `slash.convert` to `pageLink` with `pageLinkVariant: "child"`. **Link To Page** opens a native submenu and dispatches `slash.convert` with the selected `pageId` and `pageLinkVariant: "linked"`. See [pages — Page links](../architecture/pages.md#page-links) for display rules.

Title and icon edits on shipped pages use [`persistPageMetadata`](../../src/lib/pages/persist-page-metadata.ts) with a lazy seed (same as the first block edit). Optional `icon` sets emoji or `tabler:IconName` via [`persistPageIcon`](../../src/lib/pages/persist-page-icon.ts) (not a separate `page.*` command). [`PageIconPicker`](../../src/components/pages/page-icon-picker.tsx) is the shared edit surface: canvas title uses a ghost [`Button`](../../src/components/ui/button.tsx) `icon-lg` trigger (icon scale from the Button variant); sidebar rows open the same popover from overflow/context **Change icon** (anchored to the row [`SidebarMenuAction`](../../src/components/ui/sidebar.tsx), `hideTrigger`). `pageLink` rows show icons read-only via [`iconSlotClassName("default")`](../../src/components/ui/button.tsx). Emoji / Icons tabs use [`TabsList`](../../src/components/ui/tabs.tsx) `variant="line"` (auto-mounted [`TabsIndicator`](../../src/components/ui/tabs.tsx); `bg-primary` underline intersecting a full-width `border-b` divider, matching the embed link/upload tabs — [pages — Page icons](../architecture/pages.md#page-icons)). Both tabs share the composable [`GridPicker`](../../src/components/ui/grid-picker.tsx) (`h-[320px]` scroll, `h-8` search, virtualized `grid-cols-8 gap-0.5` rows) built on Base UI `Autocomplete` (`inline grid virtualized`) + [`@tanstack/react-virtual`](../../src/components/ui/grid-picker.tsx); the search is `Autocomplete.Input` rendered as our [`Input`](../../src/components/ui/input.tsx) and each cell is an `Autocomplete.Item` styled with `buttonVariants({ variant: "ghost", size: "icon-lg" })` (popover trigger stays ghost `icon-lg`). Both panels code-split: [`PageIconPickerEmojiPanel`](../../src/components/pages/page-icon-picker-emoji-panel.tsx) (emoji char) and [`PageIconPickerIconPanel`](../../src/components/pages/page-icon-picker-icon-panel.tsx) ([`TablerGlyph`](../../src/components/pages/tabler-glyph.tsx)); store each resolved type with `setPanel(() => Panel)`. [`preload-page-icon-picker.ts`](../../src/lib/pages/preload-page-icon-picker.ts) and [`WarmPageIconPickerCacheEffect`](../../src/components/pages/warm-page-icon-picker-cache-effect.tsx) idle-warm panel chunks only; `ensurePageIconPickerReady` (pointer enter / popover open) also runs `prefetchPageIconCatalogs` (TanStack Query). Open popover shows a placeholder shell until loaded (no spinners). Passive Tabler display uses SSR [`tablerIconNamesForSSR`](../../src/lib/pages/tabler-icon-names-from-pages.ts) plus by-name fetches via [`useTablerIconGlyph`](../../src/lib/pages/page-icon-catalog.ts); full catalog JSON is picker-only. Catalog data is deferred self-hosted JSON: emoji via [`emojiCatalogQueryOptions`](../../src/lib/pages/page-icon-emoji-catalog.ts) (`pnpm sync:emojibase`) and Tabler via [`tablerIconCatalogQueryOptions`](../../src/lib/pages/page-icon-catalog.ts) (`/tabler/icons.json`, `pnpm sync:icons`). [`mergePageList`](../../src/lib/pages/merge-page-list.ts) uses `local.icon ?? serverPage.icon` when overlaying a local doc. User pages have `serverBaselineHash: null`, `routeBy: "id"` (navigate on `/p/$` by metadata slug), and one empty `text` block row. Trailing empty `text` rows come from `normalizeEditablePageBlocks`. Renaming uses [`buildSlugFromTitle`](../../src/lib/pages/build-page-tree.ts) with [`dedupePageSegment`](../../src/lib/pages/build-page-tree.ts) among all siblings; the active tab updates the URL on blur via `syncUrl` / sidebar `stopRenaming` ([`syncPageUrl`](../../src/lib/pages/sync-url.ts), `userPage` when `routeBy === "id"`). Passive shipped tabs use [`useSyncPageUrl`](../../src/hooks/use-sync-page-url.ts) on `/$` only. Links use [`resolvePageNavTarget`](../../src/lib/pages/resolve-page-nav-target.ts) (`/$` vs `/p/$` by `routeBy`). Boot slug repair only (no legacy `/p/{uuid}` redirects): [pages — Route migration](../architecture/pages.md#route-migration).

Canvas structural commands also update page metadata because `localPagesCollection.blockOrder` is the durable block order for the page. The page metadata mutation and block row mutations must be accepted in the same structural transaction; see [canvas-commands](./canvas-commands.md).

## `page.reposition`

Client-only sidebar drag-and-drop lives in [`PageListContent`](../../src/components/pages/page-list.tsx) (once [`PageList`](../../src/components/pages/page-list.tsx) flips it `interactive` after hydration), wrapped in [`DndSurface`](../../src/components/dnd/dnd-surface.tsx). Transient drag state (`draggingId`, `pointer`, `dropTarget`) lives in [`createDragStore`](../../src/lib/dnd/drag-store.ts); each [`PageListItem`](../../src/components/pages/page-list-item.tsx) reads drop indicators via [`useDropTarget`](../../src/components/dnd/use-dnd.ts) so only affected rows re-render per `dragover`. Every row is a shadcn [`SidebarMenuItem`](../../src/components/ui/sidebar.tsx) + [`SidebarMenuButton`](../../src/components/ui/sidebar.tsx) inside [`SidebarMenu`](../../src/components/ui/sidebar.tsx); nested children use [`Collapsible`](../../src/components/ui/collapsible.tsx) + [`SidebarMenuSub`](../../src/components/ui/sidebar.tsx) with the rail stripped (no `SidebarMenuSubItem`/`SidebarMenuSubButton`). HTML5 DnD is on the outer `[data-page-list-row-id]` wrapper (home excluded): no grip handle; [`useDragSource`](../../src/components/dnd/use-dnd.ts) + [`usePointerClickVsDrag`](../../src/hooks/use-pointer-click-vs-drag.ts) matches the canvas gutter (quick click navigates via a custom `render` span, drag suppresses the following click). [`DragOverlay`](../../src/components/dnd/drag-overlay.tsx) + [`setEmptyDragImage`](../../src/lib/dnd/drag-image.ts) show a follow-pointer preview (icon + title, not the link URL). Chevron (`CollapsibleTrigger`) and the row overflow [`SidebarMenuAction`](../../src/components/ui/sidebar.tsx) ([`PageListRowDropdown`](../../src/components/pages/page-list-row-menu.tsx)) use `stopPropagation` on pointer down so they do not start a drag. Row indent does **not** change during drag — only after a successful drop when [`buildPageTree`](../../src/lib/pages/build-page-tree.ts) re-renders. Toolkit: [drag-and-drop](../architecture/drag-and-drop.md).

Parent rows show a chevron on the row (`CollapsibleTrigger`) that rotates when open. Drop resolution maps pointer **Y** (three-band hit testing per visible row) and **X** (horizontal unnest via [`applyPreviewDepthToDropTarget`](../../src/lib/pages/page-list-preview-depth.ts) inside [`resolvePageListDropTargetFromPointer`](../../src/lib/pages/resolve-page-list-drop-target.ts); visible rows from [`flattenVisiblePageRows`](../../src/lib/pages/flatten-visible-page-rows.ts)), then dispatches:

| Field | Meaning |
|-------|---------|
| `pageId` | Page being moved |
| `parentId` | New parent (`null` = root) |
| `insertBeforePageId` | Sibling insert anchor; omit or `null` = append last in scope |
| `appendPageLinkOnParent` | Nest drop only — append child `pageLink` on parent canvas |
| `seed` | Lazy-seed blocks for the moved page when no local doc exists |
| `parentSeed` | Lazy-seed parent page before appending `pageLink` |

[`PageListContent`](../../src/components/pages/page-list.tsx) resolves drop targets through [`DndSurface`](../../src/components/dnd/dnd-surface.tsx) (batched `dragover` on the sidebar `<nav>` via `useDropZone`), converts with [`dropTargetToRepositionCommand`](../../src/lib/pages/resolve-page-list-drop-target.ts), then awaits optional `seed` / `parentSeed` from [`loadPage`](../../src/lib/content/load-page.ts) when the moved page or nest parent has no local document yet.

Reducer path: [`pageReducer`](../../src/hooks/use-page-dispatch.ts) emits a [`page.reposition` `PageEffect`](../../src/lib/canvas/effects.ts) after [`planPageReposition`](../../src/lib/pages/reposition-page.ts) (via [`assertCanReposition`](../../src/lib/pages/reposition-page.ts)); planning errors return `{ effects: [] }` (invalid or no-op drops). Apply steps:

| Step | Implementation |
|------|----------------|
| Optional parent seed | Insert local page + `seedPageBlocks` when `parentSeed` and no local parent doc |
| Metadata write | [`persistPageReposition`](../../src/lib/pages/persist-page-reposition.ts) — update existing local row or `insert` + `seedPageBlocks` when `seed` is provided; `parentId`, `sidebarOrder`, `slug`; descendant slug updates for local descendants; [`syncPageUrl`](../../src/lib/pages/sync-url.ts) when the slug changes (`userPage` when `routeBy === "id"`) |
| Optional `pageLink` | [`appendChildPageLinkFromShard`](../../src/lib/pages/append-page-link-on-parent.ts) when `appendPageLinkOnParent` (skips duplicate links to the same `pageId`) |

Sibling order is allocated with [`computeSidebarOrderForInsert`](../../src/lib/pages/page-sidebar-order.ts) (`ORDER_STEP` sparse keys; title tiebreaker when unset). Slug allocation on create/reposition uses [`allocateUserPageSlug`](../../src/lib/pages/allocate-page-slug.ts) with [`dedupePageSegment`](../../src/lib/pages/build-page-tree.ts) among user siblings. Command conversion from drop targets: [`dropTargetToRepositionCommand`](../../src/lib/pages/resolve-page-list-drop-target.ts). Drag preview: [`DragOverlay`](../../src/components/dnd/drag-overlay.tsx). Drag payload MIME: `application/x-page-id` via [`createDragChannel`](../../src/lib/dnd/drag-channel.ts). Gestures and rules: [pages — Sidebar drag-and-drop](../architecture/pages.md#sidebar-drag-and-drop).

## Page list

`listPages` (server fn) returns metadata (including `icon`) for all `content/pages/**/*.json` pages, bundled at build time via [`page-store.server.ts`](../../src/lib/content/page-store.server.ts) (`import.meta.glob` — no filesystem reads at request time). The sidebar merges that list with user pages in `localPagesCollection` via [`mergePageList`](../../src/lib/pages/merge-page-list.ts) (local overrides, tombstones, `icon` fallback), then renders the nested tree from [`buildPageTree`](../../src/lib/pages/build-page-tree.ts) (all `parentId` scopes). The root loader prefetches `pageListQueryOptions` into React Query (`staleTime: Infinity`) to avoid sidebar flash on route changes. Sidebar rename persists through `persistPageMetadata` on each change (same lazy-seed rules as the canvas title). Drag-and-drop reparent/reorder uses `page.reposition` — see [`page.reposition`](#page-reposition) above (whole-row drag, drop-only indent). Architecture UX: [pages — Sidebar drag-and-drop](../architecture/pages.md#sidebar-drag-and-drop).

### Sidebar expand state

Chevron open/close for parent rows is a client-only UI preference (not a `page.*` command):

| Concern | Implementation |
|---------|----------------|
| Storage | `site-page-list-expanded` cookie — sorted comma-separated page ids ([`page-list-expanded-cookie.ts`](../../src/lib/pages/page-list-expanded-cookie.ts)) |
| Hydration | [`PageListContent`](../../src/components/pages/page-list.tsx) (interactive) initializes `expandedIds` from `readPageListExpandedIdsFromDocument` |
| Persistence | `useEffect` writes via `writePageListExpandedIdsToDocument` whenever `expandedIds` changes (interactive only); ids not in the merged `pages` list are dropped before write |
| Active branch | `getAncestorPageIds` + `useActivePageRef` auto-expands ancestors on navigation and merges into the same set |
| SSR | The cookie is read on the server into `PageSidebarPrefs.expandedPageIds` ([`read-page-sidebar-prefs.server.ts`](../../src/lib/pages/read-page-sidebar-prefs.server.ts) / [`load-page-sidebar-prefs.ts`](../../src/lib/pages/load-page-sidebar-prefs.ts)); the static shell renders `PageListContent` with `initialExpandedIds` so the expanded tree (plus active-page ancestors) matches after hydration — no cookie write |

See [pages — Nesting](../architecture/pages.md#nesting) for tree rules and ancestor expand behavior.

### Sidebar resize / collapse

Workspace chrome ([`PageSidebarChromeProvider`](../../src/components/pages/page-sidebar-chrome.tsx), [`PageSidebar`](../../src/components/pages/page-sidebar.tsx)) is separate from page-tree commands:

| Concern | Implementation |
|---------|----------------|
| Resize | [`PageSidebarRail`](../../src/components/pages/page-sidebar-rail.tsx) on the panel edge — **drag** resizes 12–24rem via `panel.resize()` (no visible separator handle) |
| Width storage | `site-page-sidebar-width` cookie (rem, clamped 12–24) via [`page-sidebar-layout-cookie.ts`](../../src/lib/pages/page-sidebar-layout-cookie.ts) |
| Collapsed | `site-page-sidebar-pin` cookie — `1` pinned, `0` collapsed ([`page-sidebar-pin-cookie.ts`](../../src/lib/pages/page-sidebar-pin-cookie.ts)); rail **click** collapses when pinned; rail click **pins** when shown in the hover overlay; collapsed layout has zero width |
| Hover peek | [`PageSidebarHoverReveal`](../../src/components/pages/page-sidebar-hover-reveal.tsx) — left-edge hover opens floating sidebar + scrim; closes on pointer-leave; panel uses main surface tokens (`bg-background`, accent/border) via scoped `--sidebar-*` remap on `data-page-sidebar-hover-panel` |
| Narrow viewport | [`SidebarTrigger`](../../src/components/ui/sidebar.tsx) in [`PageHeader`](../../src/components/pages/page-header.tsx) opens a Sheet (`useIsNarrowViewport`, `(max-width: 767px)`). On narrow viewports (and on desktop when the page has a [cover image](../architecture/pages.md#page-cover)) [`PageWorkspace`](../../src/components/pages/page-workspace.tsx) renders the header inside the canvas scroll region (as `headerSlot`); cover-less narrow headers scroll away; with a cover the header pins to the top with a solid `bg-background` bar while the cover scrolls up beneath it. On **desktop with a cover** the header sits below the full-bleed image (not overlaid on it) and pins with `sticky -top-16` against the desktop `md:pt-16` scroll inset. Page settings, CRUD, **Export page**, **Add/Change cover**, and activity live in [`PageHeaderMenu`](../../src/components/pages/page-header-menu.tsx) (single ⋯ trigger). Dev/sync actions live in site settings (`/settings`) — see [site-settings](../architecture/site-settings.md) |
| Shortcut | **Cmd/Ctrl+B** toggles pinned/collapsed |

See [pages — Page list](../architecture/pages.md#page-list) and [Page cover](../architecture/pages.md#page-cover). The header-menu activity summary (block / word counts — word count includes `toggleHeading` titles; `database` blocks count zero words since rows live outside the block tree, see [databases](../architecture/databases.md)) is documented in [pages](../architecture/pages.md).
